# Diagnostika-Growth — Handoff для growth-специалиста

> **Что это:** PLG-воронка (Product-Led Growth) для холодного трафика с Яндекс.Директа, конвертирующая в квалифицированный B2B-лид через бесплатную диагностику. Альтернатива классическим лид-магнитам и статьям-дополнялкам.
>
> **Основной артефакт:** `diagnostika-growth/` в репо `rmaschenko/fintablo-client-services`.
>
> **Прод-URL (GitHub Pages, временный для прогона):** [https://rmaschenko.github.io/fintablo-client-services/diagnostika-growth/](https://rmaschenko.github.io/fintablo-client-services/diagnostika-growth/)
>
> **Финальный домен:** `diagnostika.fintablo.ru` (после деплоя на FTP-инфраструктуру Финтабло).

---

## 1. Бизнес-цель и гипотеза

### 1.1 Главная цель
Получать **квалифицированные B2B-лиды** для отдела продаж Финтабло через холодный трафик с Яндекс.Директа на категориальные запросы («управленческий учёт», «финансовый учёт бизнеса», «диагностика финансов компании» и т.п.).

### 1.2 Гипотеза в одном предложении
**Бесплатная диагностика финансов с персональным разбором даёт холодному пользователю ценность сразу на экране — это создаёт мотив оставить контакт с высокой контактностью, чем статичные лид-магниты или landing «попробуйте триал».**

### 1.3 Почему PLG, а не классический лендинг
- **Холодный трафик к категории «управленческий учёт» неоднороден:** примерно 30-40% — целевые ICP, 60-70% — пограничные/anti-ICP. Лендинг «попробуйте триал» теряет 60-70% бюджета.
- **PLG-разбор даёт ценность всем:** ICP получает обоснование встречи с менеджером, не-ICP — конкретный практический разбор + триал.
- **Контактность лида выше:** клиент проходит 6 вопросов и получает разбор → форму оставляет «зная что просит» (не «получите чек-лист»).

### 1.4 Целевые KPI

| Метрика | Целевое значение | Замечание |
|---|---|---|
| CR из лендинга в `quiz_start` | 20-30% | Холодный Директ. Бенчмарк по аналогам — 15-25%. |
| CR из `quiz_start` в `quiz_complete` | 60-75% | 6 вопросов с auto-advance, без тяжёлых форм. |
| CR из `quiz_complete` в лид (форма отправлена) | 25-40% | По 3-маршрутной развилке. |
| **Доля ICP в финальных лидах** | **35% → 50%** | Главная стратегическая метрика проекта. |
| CR из ICP-лида в SQL (квалифицирован менеджером) | 60-75% | На уровне отдела продаж. |
| LTV-эффект | TBD | Замерить через 90 дней после первой когорты. |

## 2. Архитектура воронки — три слоя

### 2.1 Слой 1: Лендинг
**URL:** `/diagnostika-growth/index.html`

Цель — клик в квиз. Один CTA, реальный продукт справа в hero, минимум информационного шума.

**Структура** (CRO-оптимизированная, Pachca-подход):
1. **Hero compact** — H1 + 1 строка sub + 1 CTA + mp4 реального дашборда Финтабло справа
2. **Trust-row** под CTA — «2 300+ компаний · Интеграция с банками и системами учёта · В реестре российского ПО»
3. **Pain section** — 3 факта о российском МСБ 2025-2026 с источниками (РСПП, ФНС, исследования) — формирует контекст
4. **Симптомы** — 4 чек-листовых утверждения для самоидентификации (если ≥2 совпадает — повод пройти)
5. **Benefits** — 3 блока что будет в разборе
6. **Final CTA** — повтор главного действия с микро-доказательством

### 2.2 Слой 2: Квиз
**URL:** `/diagnostika-growth/quiz/`

Цель — собрать данные для расчёта + сегментировать на 3 маршрута.

**6 вопросов** (под ICP-критерии + расчёт + маршрутизацию):

| № | Вопрос | Зачем |
|---|---|---|
| 1 | Кто вы в компании? (собственник / финдир / бухгалтер / другое) | ICP-критерий по роли + тон разбора |
| 2 | Тип бизнеса (проектный / производство / услуги / торговля со складом) | ICP-критерий по типу + коэф. в формуле потерь |
| 3 | Годовая выручка (5 диапазонов от <30 до 500+ млн) | ICP-критерий по масштабу + база расчёта |
| 4 | Возраст бизнеса (до 3 / 3-5 / 5-10 / 10+ лет) | ICP-критерий ≥3 лет + индекс зрелости |
| 5 | Главная боль (4 варианта: маржа / разрывы / ручной свод / решения наугад) | Приоритет рекомендации + коэф. |
| 6 | Кто отвечает за финансы (CFO / финансист / бухгалтер / собственник) | Маршрутизация ICP+CFO vs ICP без CFO |

**UX:** auto-advance после клика (без лишних кнопок), прогресс-бар сверху, кнопка «Назад», localStorage с TTL 48ч (Resume-баннер).

### 2.3 Слой 3: Отчёт + 3-маршрутная развилка
**URL:** `/diagnostika-growth/quiz/report.html`

Цель — дать ценность + конвертировать в лид по правильному маршруту.

**Структура отчёта:**
- **Hero** — персонализированный заголовок по маршруту
- **01. Упущенная прибыль за год** — диапазон в рублях (3.5-9.5% годовой выручки) + декомпозиция на 3 источника, под каждым — пометка источника или метода расчёта
- **02. Индекс прозрачности учёта 0-100** — балл + зона + бенчмарк (вы / когорта / лидеры)
- **03. Что исправить в первую очередь** — narrative-flow:
  - H3 главное действие
  - Лид-параграф (что и зачем)
  - Сравнение «В Excel и таблицах» / «В Финтабло» — плоской таблицей
  - Summary-pill с обещанием срока
- **04. Финал** — единственная конверсионная точка по маршруту

### 2.4 Три маршрута сегментации

| Маршрут | Условие | Action | amoCRM |
|---|---|---|---|
| **ICP+CFO** | роль ∈ {собственник, финдир} ∧ тип ∈ {проектный, производство} ∧ выручка ≥60 млн ∧ возраст ≥3 лет ∧ есть финдир/финансист | Форма «Записаться на эксперт-разбор» | Стадия `47065159` (горячий лид) |
| **ICP без CFO** | то же ICP, но финдира нет (бухгалтер совмещает или сам собственник) | Форма «Подобрать партнёра-финансиста» (+ поле города) | Тэг `partner-needed`, отдельная стадия (TODO от Романа) |
| **Не-ICP** | любой fail из критериев ICP | CTA «Открыть Финтабло на 7 дней» — редирект на `app.fintablo.ru/register` с `utm_anti_icp=1` | НЕ отправляется в amoCRM (продуктовая воронка) |

**Логика отказа от классического anti-ICP-карманa:** не показываем «вы нам не подходите». Тёплый тон «когда дорастёте — будете готовы. Сейчас попробуйте триал». Сохраняет бренд + revenue tail через product-led воронку.

## 3. Расчётная модель разбора

### 3.1 Принцип «без галлюцинаций»
Каждое число в разборе сопровождается источником в подписи мелким шрифтом. Где нет публичной статистики — расчётные оценки на основе подтверждённых параметров (помечаем «Расчётно: …»). Диапазоны вместо точных чисел (`2,1 — 5,7 млн ₽`, не `522 156 ₽`).

### 3.2 Подтверждённые данные России 2025-2026

| Цифра | Источник |
|---|---|
| Просрочка дебиторки 8,2 трлн ₽ = 3,8% ВВП | РБК, ДП, Probankrotstvo, Фонтанка (январь 2026) |
| 31% МСБ с серьёзной просрочкой | Moscow Times, январь 2026 |
| Прибыль МСБ −10% при росте выручки +26% | Adesk через деловые СМИ, 2025 |
| 61% МСБ работают с минимальной маржой | Sber Business |
| Зарплата CFO малого бизнеса 180-250 тыс ₽/мес | HH.ru, SF Education 2025 |

### 3.3 Формулы

**Упущенная прибыль за год = выручка × коэф_боли × коэф_типа**

Декомпозиция на 3 источника потерь:
1. Время команды на ручные сверки — 0.5-1.5% выручки (расчётно по зарплате CFO МСБ × 4-5 дней закрытия × 12 мес)
2. Невидимые просадки маржи — 2-5% выручки (на основе −10% общего падения прибыли + 31% с просрочкой)
3. Задержка решений по убыточным направлениям — 1-3% выручки (оценка по сроку закрытия месяца × длительность реакции)

**Итого: 3.5-9.5% от годовой выручки.**

| Коэф_боли | Значение | Коэф_типа | Значение |
|---|---|---|---|
| Решения наугад | 1.2 | Проектный | 1.2 |
| Не знаю прибыль | 1.0 | Производство | 1.1 |
| Кассовые разрывы | 0.8 | Услуги | 1.0 |
| Ручной свод | 0.6 | Торговля | 0.8 |

**Индекс прозрачности 0-100** — функция от роли финансиста (+25 если CFO), типа бизнеса (+5 проектный), боли (-15 решения наугад), возраста бизнеса (+15 если 10+ лет). Бенчмарк когорты — расчётный профиль «тот же тип + есть финансист». Лидеры — peerScore + 20.

**Рекомендация «что исправить»** — одна из 4 по главной боли, расширена 4 полями для CRO-конверсии:
- `body` — что нужно сделать
- `whyHard` — почему трудно сделать самому
- `howFintablo` — как Финтабло закрывает это
- `proofPoint` — конкретное обещание срока

### 3.4 Тексты — через скилл fintablo-copywriter v1.1
Ключевые правила:
- Активные глаголы: подтягивает, считает, показывает, предупреждает, разносит
- Без англицизмов в основном тексте: P&L → «прибыль и убыток», workflow → «процесс»
- ОПиУ / ДДС / баланс — допустимая профессиональная терминология для ICP
- Запрет: «беспокоят», «увидите/посмотрите» в значении смотрения, «работает в связке», шаблоны конкурентов («управляйте на основе цифр», «прозрачность», «порядок»)
- Без восклицательных знаков
- Все цифры выручки — годовые («от 60 млн ₽/год»), никогда «месячные»

### 6.2 Этап 1: Тестовый Директ (10-20 тыс ₽ бюджета)
**Гипотеза:** воронка собирает квалифицированные ICP-лиды.

**Тест:** одна кампания на категориальный запрос «управленческий учёт» / «финансовый учёт компании» — десктоп Москва+Питер.

**Метрики для оценки** (после 200-500 кликов):
- CR в `quiz_start` ≥15% — сигнал что лендинг работает
- CR в `quiz_complete` ≥50% — сигнал что квиз не ломается
- CR в `lead_sent` ≥10% от `quiz_complete` — сигнал что разбор конвертирует
- Доля ICP-лидов (cfo + no_cfo) ≥30% — сигнал что ICP-сегментация попадает

**Решения по результатам:**
- Если CR в quiz_start <10% — переписать H1 / hero
- Если CR в quiz_complete <40% — упростить вопросы / сократить квиз до 5
- Если CR в lead <5% — переписать отчёт / усилить bridge к форме
- Если доля ICP <20% — пересмотреть таргетинг или критерии ICP
