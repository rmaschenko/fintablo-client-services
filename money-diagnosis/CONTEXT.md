# Контекст микросервиса «Деньги есть — прибыли нет»

## Назначение

Диагностический PLG-микросервис Финтабло. Перехватывает ICP-клиентов в момент боли «деньги на счёте есть, а прибыли нет» и через персонализированный AHA создаёт мотивацию купить Финтабло.

**JTBD (по PLG-SEO карте):** «Не понимаю где деньги» — Tier 1, приоритет #2 в глобальной roadmap.

**Флоу:** Hero + микрокалькулятор первого слоя → Квиз уточнений (8 шагов) → Персональный отчёт с суммой разрыва → Захват контакта → AmoCRM → 7-шаговая email-цепочка

**PLG-механика (модель доступа B — freemium с gate после aha):**
- **Первый слой (бесплатно):** общая сумма разрыва между деньгами на счёте и реальной прибылью
- **Второй слой (внутри Финтабло):** конкретный проект и транзакция, где утекает прибыль
- Незавершённое «хочу знать откуда конкретно» = мотивация купить

## ICP

- Проектные бизнесы: строительство, IT, агентства, производство
- Выручка 60+ млн ₽/год (5+ млн/мес)
- Есть финансист или финдиректор
- Бизнес 3+ лет

**Anti-ICP:** оборот < 30 млн, склад как центр, стартапы < 3 лет, услуги без проектной логики.

## CTA-правила (из PLG-SEO методологии)

**❌ Не работает:**
- «Попробуйте Финтабло бесплатно»
- CTA → главная страница Финтабло
- Общие формулировки без конкретики

**✅ Работает:**
- «Финтабло показывает **конкретный проект**, где утекает прибыль — автоматически каждый месяц»
- «Устал искать, где прибыль исчезает? Финтабло находит это за 2 недели вперёд»
- CTA привязан к персональному результату пользователя (подставляем его сумму разрыва)
- Gate **после** aha-момента, не до

## Tone of voice

Профессиональный CFO-тон. Термины по-взрослому (ДДС, ОПиУ, кассовый vs начислительный). Никаких emoji в контенте, никакой геймификации. Literata для заголовков = серьёзный финансовый отчёт, не квиз-игрушка.

## Технические правила

- Всё работает локально в браузере. Расчёты — JS, без внешних API.
- PHP только для лида в AmoCRM (`api/lead.php`). Токены — в `.env` (gitignore).
- Шрифты — только локальные (152-ФЗ).
- Работает на обычном российском shared-хостинге.
- Mobile-first. iPhone SE (375×667) — минимум. Кнопки касания ≥48px.

## Яндекс.Метрика

Счётчик: **61131877** (общий с growth_cfo).

**События воронки:**
- `mp_landing_view` — открытие страницы
- `mp_calc_started` — начал вводить данные в микрокалькулятор
- `mp_calc_result` — увидел первый расчёт разрыва
- `mp_quiz_step_N` (N = 1..8) — шаг квиза
- `mp_quiz_complete` — завершил квиз
- `mp_report_shown` — показан персональный отчёт
- `mp_lead_submitted` — отправлена форма
- `mp_abandoned_50+` — beforeunload на 50%+ прогрессе (эффект Зейгарник)

## Пиксельные сегменты (для ретаргета)

- **🔴 Горячий:** дошёл до результата (mp_report_shown), но не оставил контакт. Креатив: «Ты уже посчитал — вот конкретно, где утекает прибыль в твоём бизнесе»
- **🟡 Тёплый:** начал квиз, бросил на середине. Креатив: кейс с похожим бизнесом и конкретной суммой.
- **⚪ Холодный:** просмотр лендинга без действий. Исключить из ретаргета или греть контентом.
