## 1. СТРАТЕГИЧЕСКАЯ РАМКА

### Почему этот сервис запускается первым
- Самый широкий охват запросов: «управленческий учёт», «финансовый учёт бизнес», «как навести порядок в финансах», «финансовая диагностика компании» — верхний уровень воронки, максимальный объём трафика
- Покрывает оба целевых портрета (собственник + финансовый директор) в одном инструменте
- Нет сложного технического ввода — работает одинаково хорошо на мобайле и десктопе
- Квизовый формат + personalization = быстро в разработке, высокая конверсия

### Канал и трафик
- Яндекс.Директ, поисковые кампании, **только десктоп** на старте (мобайл конвертирует в 7,5 раз хуже на основном сайте — используем десктоп-таргетинг до получения данных по этому сервису)
- Целевая конверсия в лид: **8–15%** от пришедших на страницу
- После первого месяца — анализ и решение по мобайлу

## 2. ЦЕЛЕВАЯ АУДИТОРИЯ

### ICP — кто нужен (все критерии обязательны)
- Проектная бизнес-модель: каждый контракт уникален, есть этапы, обязательства до оплаты
- Выручка от 60 млн ₽/год (5+ млн/мес)
- Есть выделенный финансист или финансовый директор
- Склад не является центром управления
- Бизнес старше 3 лет
- Приоритетные отрасли: строительство / ремонт / инжиниринг, IT-проекты / разработка, агентства / маркетинг / консалтинг, производство / монтаж

### Два портрета пользователя
**Собственник:** «Не понимаю, почему прибыль есть, а денег нет. Хочу разобраться что не так с финансами в моём бизнесе»
**Финансовый директор:** «Хочу проверить, каких инструментов не хватает. Возможно, нужен специализированный инструмент вместо Excel»

### Anti-ICP — кого не брать
- Выручка до 30 млн ₽/год — не потянут продукт
- Торговля с центральным складом / оборачиваемость товара — другая бизнес-модель
- Бизнес моложе 3 лет — высокий churn
- «Всё хорошо, просто смотрю» — нет боли, нет мотивации платить
- Бухгалтеры как ЛПР (не финансисты, не собственники) — не принимают решение о покупке

### Как квалифицировать в квизе
Слайдер оборота: до 2 млн/мес → показываем альтернативный экран с CTA на регистрацию или статью. Основную воронку **не продолжаем**. В CRM лид **не отправляется**.

## 4. МЕХАНИКА КВИЗА — 10 ШАГОВ

### Порядок шагов
```
STEP_ORDER = [0, 1, 2, 3, 4, 5, 65, 10, 6, 7, 8, 9]
```
- Шаг 0: стартовый экран
- Шаги 1–5: Блок 1 «О вас и бизнесе»
- Шаг 65: промежуточный экран (AHA №1) — первые цифры, первый инсайт

**Почему гейт после шага 65, а не в конце:**

### ШАГ 7 — Главные боли (мультивыбор, макс. 2)

**Вопрос:** «Что вас беспокоит больше всего?»
**Тип:** мультивыбор, максимум 2 варианта

- Не вижу маржу по каждому проекту в моменте
- Узнаю об убытках только после закрытия проекта
- Нет понимания, когда будет кассовый разрыв
- Не могу понять, какой проект «съедает» прибыль
- Расхождение плана и факта — постоянно
- Деньги есть, но непонятно сколько реально свободно

**Инсайты по комбинациям болей:**

*«Не вижу маржу» + «узнаю об убытках после закрытия»:*
> «Это два симптома одной проблемы: учёт идёт по факту, а не в моменте. Пока проект открыт — вы управляете по интуиции»

*«Нет понимания кассового разрыва» + «деньги есть, непонятно сколько свободно»:*
> «Кассовый разрыв появляется неожиданно именно потому, что деньги на счёте не равны свободным деньгам. Авансы, обязательства, субподряд — всё это "занято" ещё до того как стало расходом»

*«Расхождение плана и факта» + «какой проект съедает прибыль»:*
> «Классика: план был, факт другой, и непонятно в каком из [N] проектов произошло отклонение. Без детализации по проектам — просто цифра»

*Любая одна боль:*
> «Это одна из главных причин, по которым к нам приходят компании вашего масштаба. Диагностика покажет, насколько типична ваша ситуация»

- Всё сразу — нужна единая система

### Классификация профиля (6 типов)

```javascript
function classifyProfile(inputs) {
  const rev = inputs.revenue
  const sys = inputs.accountingSystem
  const proj = inputs.activeProjects
  
  // Явные скобки для безопасности — && имеет приоритет выше ||
  if (sys.includes('none') || (sys.includes('excel') && proj >= 8))
    return 'blind'          // «Управление вслепую»
  
  if (sys.includes('1c') && inputs.mainProblems.includes('margin'))
    return 'accounting_illusion'  // «Бухгалтерская иллюзия»
  
  if (sys.includes('excel') && proj <= 5 && rev < 5000000)
    return 'early_stage'   // «Ранняя стадия»
  
  if (rev >= 10000000 && proj >= 10 && !sys.includes('service'))
    return 'scale_without_control'  // «Масштаб без управления» — приоритет
  
  if (sys.includes('service') && inputs.mainProblems.length <= 1)
    return 'almost_there'  // «В шаге от системы»
  
  return 'plateau'         // «Застрявшее плато»
}
```

## 11. STORAGE — localStorage и sessionStorage

```javascript
// storage.js
// UTM → sessionStorage (сбрасывается при закрытии вкладки)
// Данные квиза → localStorage (TTL 48 часов, для сценария возврата)

function captureUTM() {
  const params = new URLSearchParams(window.location.search)
  const utm = {
    source: params.get('utm_source') || '',
    medium: params.get('utm_medium') || '',
    campaign: params.get('utm_campaign') || '',
    content: params.get('utm_content') || '',
    term: params.get('utm_term') || '',
    referrer: document.referrer || '',
    pageUrl: window.location.href,
    timestamp: new Date().toISOString()
  }
  if (!sessionStorage.getItem('utm_captured')) {
    sessionStorage.setItem('diag_utm', JSON.stringify(utm))
    sessionStorage.setItem('utm_captured', '1')
  }
}

function saveQuizState(state) {
  const payload = { data: state, ts: Date.now() }
  localStorage.setItem('diag_state', JSON.stringify(payload))
}

function loadQuizState() {
  const raw = localStorage.getItem('diag_state')
  if (!raw) return null
  const payload = JSON.parse(raw)
  if (Date.now() - payload.ts > 48 * 60 * 60 * 1000) {
    localStorage.removeItem('diag_state')
    return null
  }
  return payload.data
}
```

### Сценарий возврата

При загрузке `index.html` проверяем `loadQuizState()`:
- Если есть данные → показываем баннер:
  ```
  ┌─────────────────────────────────────────┐
  │  У вас есть незавершённая диагностика    │
  │  [Продолжить с шага N]  [Начать заново] │
  └─────────────────────────────────────────┘
  ```
- Если нет → запускаем с шага 0

«Начать заново» → `localStorage.removeItem('diag_state')` + перезагрузка.
