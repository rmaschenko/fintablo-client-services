## 0. КОНТЕКСТ ДЛЯ CLAUDE CODE

Новый микросервис с нуля. Аналогов нет — не копировать ни один существующий сервис Финтабло.

**Цель:** перехватить ICP-клиентов в момент острой боли «деньги на счёте есть, а прибыли нет» — и через персонализированный AHA-момент создать мотивацию оставить заявку и купить Финтабло.

**PLG-механика:** сервис показывает первый слой правды — общую сумму разрыва между деньгами на счёте и реальной прибылью. Второй слой — конкретный проект и транзакция — только в Финтабло. Незавершённое действие «хочу знать откуда конкретно» = мотивация купить.

**Важно:** все расчёты работают локально в браузере на JavaScript. Никаких внешних API-вызовов в логике расчётов. PHP только для отправки лида в AmoCRM. Работает после выгрузки на российский хостинг без токенов и внешних сервисов.

## 1. СТРАТЕГИЧЕСКАЯ РАМКА

### Почему этот формат
Фраза «деньги есть — прибыли нет» — дословная цитата из трёх независимых кастдевов клиентов Финтабло (Сбиральников, Тюнев, Касымов). Это самая частая боль в базе из 9 интервью. Сервис воспроизводит момент AHA который клиенты описывают как причину покупки.

### Запросы для Яндекс.Директ
- «деньги есть прибыли нет»
- «почему нет прибыли при хорошей выручке»
- «выручка растёт прибыль падает»
- «куда уходит прибыль бизнеса»
- «деньги на счёте есть а бизнес убыточный»
- «как понять реальную прибыль бизнеса»
- «разница между деньгами и прибылью»

### Baseline конверсии (цель)
Текущий Директ → главная: CR АБ = 0,07%. Цель этого сервиса: 0,25–0,50% CR АБ за счёт встроенной квалификации и персонализированного AHA.

## 2. ЦЕЛЕВАЯ АУДИТОРИЯ

### ICP — кто нужен
- Проектная бизнес-модель: строительство, IT, агентства, производство
- Выручка от 60 млн ₽/год (5+ млн/мес)
- Есть выделенный финансист или финансовый директор
- Склад не центр управления
- Бизнес старше 3 лет

### Два портрета
- **Собственник:** «Оборот 150 млн, на счетах деньги есть, но прибыли нет. Что-то не так, непонятно что»
- **Финансовый директор:** «Знаю что такое кассовый разрыв, хочу быстро объяснить собственнику почему деньги ≠ прибыль на конкретных цифрах»

### Anti-ICP (мягко отсеивать)
- Выручка < 30 млн ₽/год
- Торговля со складом как центральной моделью
- Стартапы до 3 лет
- Услуги без проектной логики (парикмахерские, общепит без сети)

### 7.4 Расчётная модель (calculator.js)

```javascript
function calculate(inputs) {
  const { cashIn, receivables, expenses, balance } = inputs

  // --- Реальная прибыль (начислительный метод) ---
  // Прибыль = выручка (то что заработали) - расходы
  // Заработали = то что пришло + то что должны (дебиторка)
  const earnedRevenue = cashIn + receivables
  const realProfit = earnedRevenue - expenses

  // --- Кассовый результат (кассовый метод) ---
  // Кассовый результат = что пришло - что ушло (расходы которые оплачены)
  const cashResult = cashIn - expenses

  // --- Разрыв между кассой и начислением ---
  // Это и есть "деньги есть, прибыли нет" или наоборот
  const gap = realProfit - cashResult
  // gap > 0: реальная прибыль больше денег → деньги "заморожены" в дебиторке
  // gap < 0: деньги на счёте больше реальной прибыли → получены авансы от клиентов

  // --- Процент дебиторки от заработанного ---
  const receivablesShare = earnedRevenue > 0 
    ? Math.round((receivables / earnedRevenue) * 100) 
    : 0

  // --- Диагноз ---
  let diagnosis = ''
  let diagnosisType = ''  // 'receivables' | 'advance' | 'loss' | 'healthy'

  if (realProfit < 0) {
    diagnosisType = 'loss'
    diagnosis = 'Бизнес работает в убыток'
  } else if (receivables > cashIn * 0.3) {
    diagnosisType = 'receivables'
    diagnosis = `${receivablesShare}% заработанного "заморожено" у клиентов`
  } else if (gap < -cashIn * 0.2) {
    diagnosisType = 'advance'
    diagnosis = 'На счёте есть деньги клиентов — ещё не заработанные'
  } else {
    diagnosisType = 'healthy'
    diagnosis = 'Разрыв в пределах нормы'
  }

  // --- Годовая экстраполяция ---
  const annualGap = Math.abs(gap) * 12

  // --- Отраслевой контекст (для персонализации) ---
  const industryContext = getIndustryContext(inputs.industry, receivablesShare, diagnosisType)

  return {
    earnedRevenue,
    realProfit,
    cashResult,
    gap,
    receivablesShare,
    diagnosisType,
    diagnosis,
    annualGap,
    industryContext
  }
}
```

## 17. UTM-ПАРАМЕТРЫ — ЗАХВАТ И ХРАНЕНИЕ

```javascript
// storage.js — захват UTM при первом входе
// UTM хранится в sessionStorage (сбрасывается при закрытии вкладки)
// Данные квиза хранятся в localStorage (48 часов для сценария возврата)

function captureUTM() {
  const params = new URLSearchParams(window.location.search)
  const utm = {
    source: params.get('utm_source') || '',
    medium: params.get('utm_medium') || '',
    campaign: params.get('utm_campaign') || '',
    content: params.get('utm_content') || '',
    term: params.get('utm_term') || '',
    referrer: document.referrer || '',
    pageUrl: window.location.href,
    timestamp: new Date().toISOString()
  }
  
  // Сохраняем в sessionStorage (UTM — только для текущей сессии)
  if (!sessionStorage.getItem('utm_captured')) {
    sessionStorage.setItem('fintablo_utm', JSON.stringify(utm))
    sessionStorage.setItem('utm_captured', '1')
  }
}
captureUTM()

// Данные квиза — localStorage с TTL 48 часов
function saveQuizState(state) {
  const payload = { data: state, ts: Date.now() }
  localStorage.setItem('moneyprofit_state', JSON.stringify(payload))
}

function loadQuizState() {
  const raw = localStorage.getItem('moneyprofit_state')
  if (!raw) return null
  const payload = JSON.parse(raw)
  const age = Date.now() - payload.ts
  if (age > 48 * 60 * 60 * 1000) {
    localStorage.removeItem('moneyprofit_state')
    return null
  }
  return payload.data
}
```
