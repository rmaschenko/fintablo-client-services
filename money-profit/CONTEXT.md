# Контекст микросервиса «Деньги есть — прибыли нет»

## Назначение
Диагностический PLG-микросервис Финтабло. Перехватывает ICP-клиентов в момент боли «деньги на счёте есть, а прибыли нет» и через персонализированный AHA создаёт мотивацию купить Финтабло.

**Флоу:** Лендинг с микро-калькулятором → Квиз (уточнения) → Персональный отчёт с суммой разрыва → Захват контакта → AmoCRM

**PLG-механика:** показываем **первый слой правды** — сумму разрыва между деньгами и прибылью. **Второй слой** (конкретный проект/транзакция) — только внутри Финтабло. Незавершённое = мотивация.

## Источники истины

- **ТЗ:** [TZ.md](./TZ.md) — полное техническое задание
- **DS:** Fintablo Design System Kit v1.0 (IBM Plex Sans/Mono + Literata, токены в `css/style.css`)
- **UX-паттерны:** Quiz UX Patterns Guide (иллюзия прогресса, микро-награды, эффект Зейгарник)
- **Бренд-ориентир:** `~/Вайбкод/fintablo/` (growth_cfo) — общие вещи типа cookie-баннера, футера с реквизитами ООО «Нескучный Финансовый Софт», Яндекс.Метрика (счётчик 61131877)

## ICP

- Проектные бизнесы: строительство, IT, агентства, производство
- Выручка 60+ млн ₽/год (5+ млн/мес)
- Есть финансист или финдиректор
- Бизнес 3+ лет

**Anti-ICP** (мягко отсеивать): оборот < 30 млн, склад как центр, стартапы < 3 лет, услуги без проектной логики.

## Tone of voice

Профессиональный CFO-тон. Термины по-взрослому (ДДС, ОПиУ, кассовый vs начислительный). Никаких emoji в контенте, никакой геймификации. Literata для заголовков = серьёзный финансовый отчёт, а не квиз-игрушка.

## Технические правила

- Всё работает локально в браузере. Расчёты — JS, без внешних API.
- PHP только для лида в AmoCRM (`api/lead.php`). Токены — в `.env` (gitignore).
- Шрифты — только локальные. Google Fonts / любые CDN запрещены (152-ФЗ).
- Сайт должен работать на обычном российском shared-хостинге после выгрузки по FTP.
- Mobile-first. iPhone SE (375×667) — минимум. Кнопки касания ≥ 48px.

## Яндекс.Метрика

Счётчик: **61131877** (тот же, что у growth_cfo — агрегируем в одном отчёте).

Обязательные события:
- `quiz_start` — открыт первый экран
- `quiz_step_N` — N = 1..8, каждый шаг
- `quiz_complete` — пройдены все вопросы
- `result_shown` — показан персональный расчёт
- `lead_submitted` — отправлена форма
- `quiz_abandoned` — beforeunload на середине (для воронки выхода)

## AmoCRM

Настройка в конце разработки. Та же учётка, что у growth_cfo, воронка — отдельная (будет создана позже). До получения токенов `api/lead.php` работает в CSV-backup режиме (пишет в `leads_backup.csv` рядом).
