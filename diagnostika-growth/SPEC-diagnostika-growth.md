## 0. Контекст и цели

### 0.1 Что строим
Конверсионный лендинг с квизом «Финансовая диагностика» для холодного трафика Яндекс.Директа. Архитектура — два экрана:

- **Экран 1 (этот лендинг):** hero, pain-блок, симптомы, CTA в квиз. Цель — клик в квиз.
- **Экран 2 (твой существующий микросервис):** квиз, расчёт финансового профиля, AHA-экран с дашбордом, gate-форма, маршрутизация по ICP. Цель — квалифицированный лид в amoCRM.

### 0.2 Главные KPI
1. CR из лендинга в `quiz_start` (целевой канал — Директ)
2. CR из `quiz_start` в `quiz_complete` (контакт захвачен)
3. **Доля ICP в финальных лидах** (главная метрика проекта — рост с 35% до 50% по канону `FINTABLO_UTP_LAUNCH`)
4. CR из ICP-маршрута в SQL «АБ-квалификация» (стадия 47065159 amoCRM)

### 0.3 Что лендинг делает специально
- Не отсекает anti-ICP на лендинге — отсечение работает в квизе через 3-маршрутную развилку
- Не конкурирует с РК «Экспресс-разбор» (`FINTABLO_EXPRESS_RAZBOR_LAUNCH.md`) — финальный CTA для ICP объединяется с экспресс-разбором (Архитектура A)
- Соблюдает 152-ФЗ (хостинг РФ, явное согласие) и закон о русском языке (никаких иноязычных терминов без перевода)

## 8. Связь с микросервисом-квизом

### 8.1 Архитектура переходов

```
Пользователь → Директ → diagnostika.fintablo.ru/  (этот лендинг)
                       ↓ клик «Пройти диагностику»
                       diagnostika.fintablo.ru/quiz/  (твой микросервис)
                       ↓ маршрутизация по ICP
                       ├── ICP → запись на экспресс-разбор
                       ├── partial fit → app.fintablo.ru/register
                       └── anti-ICP → email + Excel-шаблон
```

### 8.2 Что должно передаваться через URL при переходе

```
?utm_source=...
&utm_medium=...
&utm_campaign=...
&utm_term=...
&utm_content=...
&yclid=...      // Yandex Click ID
&_ym_uid=...    // Метрика UID
&from=landing   // флаг что пришёл с лендинга, не напрямую
```

Все UTM захватываются на лендинге → передаются через querystring в микросервис → микросервис сохраняет в `localStorage` (TTL 48ч) → подкладывает в payload при отправке в amo.

### 8.3 Что меняется в микросервисе (отдельная итерация после Hero)

Зафиксировано как известный TODO для следующей итерации, не для этого спринта:
- 3-маршрутная развилка ICP в `submit()`

## 9. Визуальная сетка и компоненты

### 9.1 Платформа

- Vanilla HTML/CSS/JS, **без фреймворков** (синхронно с твоим микросервисом)
- Один HTML-файл (как у тебя на GitHub Pages)
- Стили inline или в `<style>` — без отдельных CSS-файлов
- Шрифты: системные стек (`-apple-system, ...`) — без подгрузки веб-шрифтов (скорость загрузки)

### 9.2 Цветовая система — наследуем от микросервиса

Использовать те же CSS-переменные, что в твоём `money-diagnosis/`. Если в микросервисе их нет в виде переменных — извлечь и зафиксировать в `:root`. **Не выдумывать новые цвета.**

### 9.3 Типографика

- H1 — 48–56px desktop, 32–36px mobile
- H2 — 32–36px desktop, 24–28px mobile
- Body — 17–18px desktop, 16px mobile
- Цифры в hero-карточке — моноширинный шрифт (`SF Mono`, `JetBrains Mono`, fallback `monospace`)

### 9.4 Контейнер

- Max-width: 1200px (как на fintablo.ru)
- Padding по бокам: 24px desktop, 16px mobile

### 9.5 Минимум анимаций

- Карусель кейсов: fade-out / fade-in 400ms
- Hover на CTA: micro-elevation `translateY(-2px)`
- Никаких параллаксов, scroll-triggered animations, marquee

### 9.6 Мобайл

- Responsive до 480px
- На мобайле: карусель кейсов скрывается, hero становится одно-колоночным
- **Warning-баннер для мобайла:** `display: block` только на mobile breakpoint


### 10.2 Цели Метрики (отправлять `ym(61131877, 'reachGoal', 'NAME')`)

| Событие | Goal Name |

| Переход в квиз (любой CTA) | `go_to_quiz` |

### 10.3 UTM-захват

JavaScript-сниппет в `<head>` после Метрики:

```javascript
(function() {
  const params = new URLSearchParams(window.location.search);
  const utmKeys = ['utm_source','utm_medium','utm_campaign','utm_term','utm_content','yclid'];
  const captured = {};
  utmKeys.forEach(k => {
    const v = params.get(k);
    if (v) captured[k] = v;
  });
  if (Object.keys(captured).length) {
    captured._captured_at = Date.now();
    localStorage.setItem('ft_utm', JSON.stringify(captured));
  }
})();
```

Все клики по CTA в квиз дописывают сохранённые UTM в querystring перехода.
