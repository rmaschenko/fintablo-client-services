## Smoke-test (быстрая проверка после деплоя)

```bash
# 1. Все страницы отдают 200
for path in / /quiz/ /quiz/report.html /quiz/thankyou.html; do
  echo -n "$path: "
  curl -sI "http://localhost:8765/diagnostika-growth$path" | head -1
done

# 2. Все JS-файлы синтаксически валидны
cd diagnostika-growth/quiz/js
for f in *.js; do node --check "$f" && echo "✓ $f"; done

# 3. 12 шаблонов отдают 302 на disk.yandex.ru
node -e "global.window={};require('./templates.js');
  Object.values(global.window.Templates.REGISTRY)
    .forEach(t => console.log(t.id + '\\t' + t.url))" | \
  while IFS=$'\t' read id url; do
    code=$(curl -sI -A 'Mozilla/5.0' "$url" -o /dev/null -w "%{http_code}")
    echo "$code $id"
  done
# Ожидаем: 302 для всех 12

# 4. lead.php валидирует на пустой POST
curl -X POST http://localhost:8765/diagnostika-growth/quiz/api/lead.php \
  -H "Content-Type: application/json" -d '{}'
# Ожидаем JSON: {"error":"name and phone required"}
# (на python3 -m http.server PHP не работает — этот тест только для FTP-сервера)
```

## Чек-лист совместимости браузеров

Прогнать Сценарий 1 (Hot ICP) в каждом:

| Браузер | Ожидаемое поведение |
|---|---|
| Chrome 120+ | Все работает, gauge SVG анимируется |
| Safari 16+ (iOS + macOS) | Шрифт Inter рендерится корректно, форма работает |
| Firefox 120+ | Все работает |
| Edge 120+ | Все работает |
| Mobile Safari (iPhone) | Quiz auto-advance работает, грид шаблонов 1 колонкой |
| Chrome Mobile (Android) | То же |

### Известные особенности
- Safari < 14 — не поддерживается (`AbortController` нужен в lead.js)
- Firefox с включённой защитой трекинга может блокировать Я.Метрику — это нормально

## Чек-лист доступности (a11y)

- [ ] Все интерактивные элементы (`button`, `a`) с `tabindex` — клавиатурная навигация работает
- [ ] Контрастность текста ≥ 4.5:1 (проверять Lighthouse)
- [ ] `<label>` привязаны к `<input>` через `for` или вложенностью
- [ ] `aria-hidden="true"` на декоративных SVG (визуализации)
- [ ] Skip-link `.skip-link` работает (Tab → «Перейти к содержимому»)

## Тестирование 152-ФЗ согласия

- [ ] Чекбокс согласия не предзаполнен (пустой по умолчанию)
- [ ] Кнопка submit disabled до отметки чекбокса
- [ ] При отмеченном чекбоксе и сабмите в payload приходит:
  ```json
  "consent": {
    "given": true,
    "timestamp": "ISO 8601",
    "policyUrl": "https://fintablo.ru/position",
    "offerUrl": "https://fintablo.ru/oferta",
    "operator": "ООО «Нескучный финансовый софт», ИНН 2311303019"
  }
  ```
- [ ] Рекламный чекбокс — отдельный, не обязательный, передаётся как `marketingConsent: bool`
- [ ] Ссылки на «Политику» и «Оферту» открываются в новой вкладке (`target="_blank"`)
