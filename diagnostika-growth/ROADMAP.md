# diagnostika-growth — ROADMAP отложенных доработок

Документ фиксирует доработки, которые **отложены** до появления соответствующих ресурсов (CRM-маркетолог, email-инфраструктура, аналитик). Приоритет — по ожидаемому ROI и сложности подключения.

---

## Контекст текущего MVP

На момент 2026-05-07 проект запущен в режиме **MVP-Lite без email-сервиса**:
- **Anti-ICP**: на success-странице сразу 3 прямые ссылки на скачивание шаблонов с Я.Диска **без формы и без email-gate**. Бонус для торговли со складом — 4-я ссылка (Управление запасами).
- **Warm ICP**: редирект на `app.fintablo.ru/register?utm_campaign=warm_partial_fit` — продуктовая воронка Финтабло.
- **Hot ICP / Hot ICP без ФФ**: лид-форма (имя + телефон + email + 152-ФЗ + рекламное согласие) → `api/lead.php` → amoCRM.
- **Pixel-сегментация Я.Метрики ✅ сделано** (события `dg_anti_icp_segment`, `dg_warm_icp_segment`, `dg_hot_icp_segment`, `dg_lead_hot_icp`, `dg_lead_hot_icp_no_finance`, `dg_warm_icp_click`, `dg_anti_icp_download` — все с params: `business_type`, `revenue`, `pain`, `cfo_status`, `anti_subtype`).

Что **не входит в MVP** и наслоится поверх:
- Автоматическая email-доставка шаблонов (для anti-ICP)
- Welcome-цепочки и триггеры между сегментами
- Когортная аналитика и A/B-тесты на форму

Эти доработки требуют CRM-маркетолога и/или интеграции с email-сервисом — и **не требуют переписывания фронта**, накатываются поверх существующей инфраструктуры.

## Доработки (по приоритету)

### P1 — Email-доставка шаблонов

**Что:** при сабмите формы anti-ICP отправлять автоматическое письмо с темой «Ваши шаблоны от Финтабло» и ссылками на 3 файла в теле письма.

**Зачем:** сейчас direct links живут только на success-page. Если человек закроет вкладку — потеряет доступ. Email = страховка + повод для retention.

**Что нужно:** подключение к одному из сервисов рассылок Финтабло (Mindbox / UniSender / SendPulse / Unisender Go). Транзакционный шаблон письма.

**Срок реализации:** 1-2 дня после получения доступа к сервису.

---

### P1 — Welcome-серия для anti-ICP (drip campaign)

**Что:** 5-точечная цепочка касаний для anti-ICP-сегмента после получения шаблонов:

| День | Тема | Содержание |
|---|---|---|
| 0 | Шаблоны внутри | Доставка 3 шаблонов + бонус |
| 7 | Как разобрать ОПиУ за вечер | Образовательный контент по выбранному шаблону |
| 30 | Что меняется на выручке 30+ млн | Первый сигнал об экспансии |
| 60 | Кейс: как X вырос с 20 до 60 млн | Социальное доказательство роста |
| 90 | Готовы попробовать Финтабло? | Re-quiz CTA → возможный переход в Warm/Hot |

**Зачем:** превратить anti-ICP-лид в будущий warm/hot. По расчёту 5-10% дорастают за 6-12 месяцев — без re-engage этот пул теряется.

**Что нужно:** CRM-маркетолог + email-сервис.

---

### P2 — Триггер «дорос до 30 млн»

**Что:** автоматическая отметка в БД, когда anti-ICP-лид сообщает о росте выручки. Источники сигнала:
- Перепрохождение квиза (повторный visit + новые ответы)
- Клик на CTA «Готовы попробовать Финтабло» в email-цепочке
- Self-reported апдейт (форма «обновить данные» в письме)

При срабатывании триггера: автоматический переход лида в pipeline Warm ICP, уведомление менеджеру в amoCRM.

**Зачем:** автоматическое извлечение ценности из БД anti-ICP без ручной работы менеджера.

**Что нужно:** CRM-маркетолог + amoCRM-инженер для настройки воркфлоу.

---

### ✅ P2 — Pixel retargeting Я.Директ — СДЕЛАНО (2026-05-07)

В коде `report.js` срабатывают сегментные goals на показ финального блока:
- `dg_anti_icp_segment` — для exclusion из активной кампании
- `dg_warm_icp_segment` — для soft-retargeting (creative «попробуй продукт»)
- `dg_hot_icp_segment` — для look-alike + усиленный ретаргет
- `dg_lead_hot_icp` / `dg_lead_hot_icp_no_finance` — для look-alike по горячим лидам
- `dg_warm_icp_click` — клик на CTA «Попробовать Финтабло» (warm)
- `dg_anti_icp_download` — клик на скачивание шаблона (с параметром `template`)

Ко всем событиям передаются params: `business_type`, `revenue`, `pain`, `cfo_status`, `anti_subtype` — позволяет строить детальные сегменты в Я.Метрике.

**Что осталось сделать в кабинете** (не в коде):
1. В Я.Метрике создать цели по этим именам событий
2. В Я.Директе из целей создать аудитории
3. Настроить exclusion для `dg_anti_icp_segment` и look-alike для `dg_hot_icp_segment` + `dg_lead_*`

### P2 — Сегментация anti-ICP по причине

**Что:** разделить БД anti-ICP на:
- **anti-icp-trade** (торговля со складом — структурный блокер навсегда)
- **anti-icp-services** (услуги без проектной модели — структурный блокер)
- **anti-icp-small** (выручка <30 млн — временный блокер)

Для каждой когорты — свой контент в email-цепочке. Trade и Services — упор на образовательный контент по их специфике, без push на Финтабло (бессмысленно). Small — упор на рост и Финтабло как точка дорастания.

**Зачем:** релевантность контента → выше open/click rate, ниже unsubscribe.

**Что нужно:** CRM-маркетолог.

---

### P3 — Cohort retention dashboard

**Что:** дашборд в Datalens (или Looker) с метриками anti-ICP-когорты:
- % дорастающих в Warm/Hot за 6 / 12 / 18 мес
- Open rate / click rate по поинтам цепочки
- Top-болезненные шаблоны (какие чаще скачивают)
- LTV anti-ICP-конвертированных лидов vs прямой воронки

**Зачем:** управление ROI воронки anti-ICP. Сейчас работаем «вслепую» — нет данных об эффективности.

**Что нужно:** аналитик + доступ к amoCRM API.

---

### P3 — A/B тесты на email-gate

**Что:** тестировать варианты:
- 1 поле (email) vs 2 поля (email + имя)
- Direct link vs «отправим на email»
- 3 шаблона vs 2 шаблона vs 1 шаблон под боль

**Зачем:** калибровка conversion rate на лид-магните.

**Что нужно:** A/B-тест-инструмент (GrowthBook / Я.Метрика эксперименты) + аналитик для интерпретации.

---

### P3 — Расширение библиотеки шаблонов

**Что:** добавить шаблоны под специфические сегменты anti-ICP, которых сейчас нет в выдаче:
- Для общепита (даже не из библиотеки Финтабло — можно лицензировать у iiko/R-Keeper-партнёров)
- Для бьюти-сегмента (POS-учёт, абонементы, ежедневная выручка)
- Для микробизнеса <10 млн (упрощённая управленка на 1 листе)

**Зачем:** релевантность под отраслевые anti-ICP — выше perceived value, выше open rate в email-цепочке.

**Что нужно:** CRM-маркетолог + content-marketer + (возможно) партнёрские договорённости.
