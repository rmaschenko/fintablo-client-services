# Контекст микросервиса «Диагностика финансового учёта» — Финтабло

## Назначение
PLG-микросервис для платного трафика Яндекс.Директ. 10-шаговый квиз с промежуточным AHA-экраном, гейтом контакта после AHA и персональным отчётом из 5 секций. Генерирует квалифицированные B2B-лиды.

**Флоу:** Квиз (0→5) → AHA-экран (65) → Гейт контакта (10) → Вопросы (6→9) → Отчёт → Thankyou.

**Прод:** `fintablo.ru/tools/diag/financial/`

---

## ICP
Проектный бизнес (стройка / IT / агентство / производство) с выручкой от 60 млн ₽/год, выделенный финансист, бизнес старше 3 лет.

Два портрета: **собственник** («не понимаю где деньги») и **финансовый директор** («хочу проверить, каких инструментов не хватает»).

## Anti-ICP (soft gate)
При обороте < 2 млн/мес — альтернативный экран с CTA на регистрацию / кейс. В CRM **не отправляем**.

## Механика квиза

**STEP_ORDER:** `[0, 1, 2, 3, 4, 5, 65, 10, 6, 7, 8, 9]`

Ключевой паттерн: гейт **после** AHA-момента (шаг 65 показывает первые цифры — мотивация получить полный отчёт). После шага 9 — автоматический redirect на `report.html` (полный режим).

Два именованных блока прогресса:
1. «О вас и бизнесе» (шаги 1–5)
2. «Финансовый контроль» (шаги 6–9)

Персонализация по `role` (owner / financier) — разные инсайты, формулировки, CTA.

Шаг 9 показывается **только** при `role === 'owner'`.

## Расчётная модель

14 вычисляемых метрик (см. `calculator.js`):
- Базовые: `monthlyRevenue`, `annualRevenue`
- Коэффициенты риска по системе учёта (max из выбранных)
- `estimatedAnnualLoss`, `riskProjectsCount`, `transparencyIndex` (0–100)
- `profileType` — 6 типов: `blind`, `accounting_illusion`, `early_stage`, `scale_without_control`, `almost_there`, `plateau`
- `primaryRiskZone` — по отрасли + системе
- `icpScore` (0–100) → `lead_A` / `lead_B` / `lead_C`

## Отчёт (report.html)

**Режим превью** (без контакта): секции 1–2 видны, 3–5 под blur + замок.
**Режим полный** (после отправки формы): все 5 секций.

1. Финансовый профиль (3 метрики + карточка профиля)
2. Где теряется (3 персонализированные карточки)
3. Вопросы без ответа (за гейтом)
4. Сценарии роста (4 варианта, рекомендован Финтабло)
5. Финтабло для вашей ситуации (5 функций + 2 кейса + CTA на thankyou)

**CTA на report.html:** «Записаться на бесплатный разбор» → `thankyou.html` (форма уже отправлена на шаге 10).

## Лидогенерация

### Поля лида (25+)
Все ответы квиза + вычисленные метрики + UTM + referrer + pageUrl + timestamp + sessionDuration.

### CSV Backup
`api/leads/fin_diagnostics_YYYY-MM.csv`, BOM UTF-8, разделитель `;`.

### AmoCRM
- Домен `tablo2.amocrm.ru`, pipeline `5278171`, stage `47065159`
- Теги: `micro_service`, `fin_diagnostics`, `direct_client` + квалификация по `icpScore`
- Название лида: `Диагностика: [Имя] — [Отрасль] — [Профиль] — [Оборот]`
- Honeypot поле `website`
