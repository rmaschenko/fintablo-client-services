# ТЗ: Микросервис «Диагностика финансового учёта»

**Приоритет запуска:** 1 (первый).
**Владелец:** Роман (маркетинг + PR, Финтабло).
**Целевой URL:** fintablo.ru/tools/diag/financial/
**Канал:** Яндекс.Директ (только десктоп на старте).

## Цель сервиса
Генерировать квалифицированные B2B-лиды для отдела продаж Финтабло. Человек проходит диагностику → получает персональный отчёт → оставляет телефон → менеджер звонит.

## ICP (все критерии обязательны)
- Проектная бизнес-модель (контракты, этапы, обязательства до оплаты)
- Выручка от 60 млн ₽/год (5+ млн/мес)
- Есть выделенный финансист или финансовый директор
- Склад не является центром управления
- Бизнес старше 3 лет
- Отрасли: стройка / IT / агентство / производство

## Anti-ICP (мягко отсекаем)
- Выручка < 30 млн ₽/год — не поймут продукт
- Торговля с центральным складом — другая модель
- Младше 3 лет — churn
- «Всё хорошо, просто смотрю» — нет мотивации
- Бухгалтеры как ЛПР — не принимают решение

При обороте < 2 млн/мес показываем альтернативный экран (регистрация / кейс), в CRM не отправляем.

## Структура квиза — 10 шагов

**STEP_ORDER:** `[0, 1, 2, 3, 4, 5, 65, 10, 6, 7, 8, 9]`

Гейт (форма контакта) стоит на шаге 10 — **после** промежуточного AHA-экрана (шаг 65), **до** вопросов 6–9. После шага 9 — редирект на `report.html`.

- **0** — Стартовый экран (H1, social proof, 3 буллета, CTA)
- **1** — Роль (собственник / финансист) — ключевое, определяет весь нарратив
- **2** — Имя
- **3** — Отрасль (5 карточек)
- **4** — Оборот (слайдер, дискретные зоны 500к–100м)
- **5** — Кол-во активных проектов (слайдер 1–30)
- **65** — Промежуточный AHA-экран (мини-дашборд: потери, индекс прозрачности, зона риска)
- **10** — Захват контакта (PAS-заголовок, превью отчёта с blur, форма)
- **6** — Система учёта (мультивыбор)
- **7** — Главные боли (мультивыбор, макс. 2)
- **8** — Желаемый результат
- **9** — Финансист в штате (только для role=owner)

## Прогресс-бар
Два блока: «О вас и бизнесе» (1–5) и «Финансовый контроль» (6–9).

## Расчётная модель (14 метрик)
- `monthlyRevenue`, `annualRevenue`
- `riskCoeff` — max из выбранных систем учёта: none=0.20, excel=0.13, 1c=0.10, other=0.08, service=0.04
- `estimatedAnnualLoss = annualRevenue × riskCoeff`
- `riskShare` — max из riskProjectShare: none=0.50, excel=0.30, 1c=0.25, other=0.20, service=0.10
- `riskProjectsCount = round(activeProjects × riskShare)`
- `transparencyIndex` (0–100) = max(5, min(95, baseScore + scaleBonus − problemPenalty))
- `profileType` — 6 типов (blind / accounting_illusion / early_stage / scale_without_control / almost_there / plateau)
- `primaryRiskZone` — по отрасли + системе
- `icpScore` (0–100) → lead_A (61+), lead_B (31–60), lead_C (0–30)

## Отчёт (report.html)
5 секций, персонализированные:
1. Финансовый профиль (3 метрики + профиль)
2. Где теряется (3 карточки по отрасли + проблемам)
3. Вопросы без ответа (персонализированы по профилю)
4. Сценарии (4 варианта, рекомендован Б — Финтабло)
5. Финтабло для вашей ситуации (5 функций + 2 кейса + CTA)

Режимы: **превью** (без контакта) — секции 1–2, остальные blur + замок. **Полный** — все 5 секций.
